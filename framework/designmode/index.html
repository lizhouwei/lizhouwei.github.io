# 设计模式组成

  • 设计模式名称：描述模式的名称
  • 环境和问题：描述在什么场景下，出现什么样的特定问题
  • 解决方案：描述通过某种设计模式解决问题
  • 效果：描述通过设计模式产生的效果，以及引入的新问题

# 设计模式分类

  • 创建型模式：抽象了对象实例化过程，用来帮助创建对象实例
  • 结构型模式：描述如何组合类和对象以获得更大结构
  • 行为型模式：描述算法和对象之间职责的分配
    创建型 结构型 行为型
    Factory Method（工厂方法）
    Abstract Factory（抽象工厂）
    Builder（构建者）
    Facade（外观模式）
    Adapter（适配器模式）
    Singleton（单例）
    Property（原型）

# 设计模式大法等级

  1. 基本入门级
    能够正确理解和掌握每个设计模式的基本知识，能够识别代码设计中使用的设计模式。

  2. 基本掌握级
    能够结合实际应用场景，根据场景，分析出现了什么样的问题，采用何种模式来解决它。

  3. 掌握级
    能够结合实际应用场景，能对设计模式进行变种使用。

  4. 熟练级
    能够从思想上吸收设计模式的精髓，并融入到自己的思路中。在代码开发时能够随心所欲，信手拈来,无招胜有招。
