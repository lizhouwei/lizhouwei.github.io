# 矩阵最小路径和

# 题目

给定一个矩阵m, 从左上角开始每次只能向右或者向下走,最后到达右下角的位置,路径上所有的数值累加起来就是路径和,返回所有的路径中最小的路径和.

# 举例

如果给定的矩阵m如下：

[1359813450618840]\begin{bmatrix} 1 & 3 & 5 & 9 \\ 8 & 1 & 3 & 4 \\ 5 & 0 & 6 & 1 \\ 8 & 8 & 4 & 0 \end{bmatrix} ​⎣​⎢​⎢​⎡​​​1​8​5​8​​​3​1​0​8​​​5​3​6​4​​​9​4​1​0​​​⎦​⎥​⎥​⎤​​

路径1，3，1，0，6，1，0是所有路径中路径和最小的，所以返回12。

# 解析

  • 最终问题f(i,j)：
    假设从左上角到达第m[i,j]的最短路径为f(i,j)。
  • 最优子结构：
    到达m[i,j]位置时，最短路径有两种可能，第一种可能是第 m[i-1][j] + m[i][j]，第二种可能是 m[i][j-1] +m[i][j]。
  • 最优子结构与最终问题之间的关系：
    按照上述表达，那么可以归纳出f(i,j) = [i][j] + min{ [i-1][j] , [i][j-1] } (i>=0)
    如果位置是在第一行时，则最短路径只有m[0][j-1]，即f(i,j) = m[0][j-1] + m[0][j] ;
    如果位置是在第一列时，则最短路径只有m[i][0]，即f(i,j) = m[i-1][0] + m[0][j] ;
    如果位置是在m[0][0]时，则最短路径只有m[0][0]，即f(i,j) = m[0][0] ;完整公式如下：

f(i,j)={m[i][j]i=0,j=0m[0][j−1]+m[0][j]i=0m[i−][0]+m[0][j]j=0min{m[i−1][j],m[i][j−1]}+m[i][j]i≥1,j≥1 f(i,j) = \begin{cases} m[i][j] & i=0,j=0 \\ m[0][j-1] + m[0][j] & i = 0 \\ m[i-][0] + m[0][j] & j=0 \\ min\{m[i-1][j],m[i][j-1]\}+ m[i][j] & i \geq 1 , j \geq 1 \end{cases}f(i,j)=​⎩​⎪​⎪​⎪​⎨​⎪​⎪​⎪​⎧​​​m[i][j]​m[0][j−1]+m[0][j]​m[i−][0]+m[0][j]​min{m[i−1][j],m[i][j−1]}+m[i][j]​​​i=0,j=0​i=0​j=0​i≥1,j≥1​​

# 暴力搜索方法解答

public int minPathSum(int[][] m) {
    if (m == null || m.length == 0) {
        return 0;
    }
    return process(m, m.length - 1, m[0].length - 1);
}

public int process(int[][] m, int row, int col) {
    if (row > m.length && col > m[0].length) {
        return 0;
    }
    int res = 0;
    if (row == 0 && col == 0) {
        return m[row][col];
    }
    if (row == 0) {
        res = process(m, 0, col - 1);
    } else if (col == 0) {
        res = process(m, row - 1, 0);
    } else {
        res = Math.min(process(m, row - 1, col), process(m, row, col - 1));
    }
    res += m[row][col];
    return res;
}

TIP

过暴力递归求解的方式很明显的一个缺点就是子问题被重复计算了多遍，时间复杂度为指数级，性能低下,时间复杂度为 O(N2)\ O(N^2)O(N​2​​)。

# 记忆搜索方法解答

public int minPathSum(int[][] m) {
    if (m == null || m.length == 0) {
        return 0;
    }
    int[][] map = new int[m.length][m[0].length];
    return process(m, m.length - 1, m[0].length - 1, map);
}

public int process(int[][] m, int row, int col, int[][] map) {
    if (row > m.length && col > m[0].length) {
        return 0;
    }
    int mapValue = map[row][col];
    if (mapValue != 0) {
        return mapValue;
    }
    int res = 0;
    if (row == 0 && col == 0) {
        return m[row][col];
    }

    if (row == 0) {
        res = process(m, 0, col - 1, map);
    } else if (col == 0) {
        res = process(m, row - 1, 0, map);
    } else {
        res = Math.min(process(m, row - 1, col, map), process(m, row, col - 1, map));
    }
    res += m[row][col];
    map[row][col] = res;
    return res;
}

TIP

求解方式相比暴力递归方式有了时间复杂度上的优化,时间复杂度为 O(N2)\ O(N^2)O(N​2​​)。

# 动态规划方法解答

假设矩阵m的大小为MXN，行数为M,列数为N，先生成大小和矩阵m一样的矩阵dp，dp[i][j]的值表示从左上角m[0，0] 到位置m[i，j]的最小路径和，对于m的第一行所有位置来说，即m[0，0..N-1]，从m[0，0]走到m[0,j]位置的路径和就是m[0][0..j]的所有位置的最小路径和累加的结果。同理，对于矩阵m的第一列所有位置来说，即m[0..M-1，0]，从m[0，0]走到m[j,0]位置的路径和就是m[0..j][0]的所有位置的最小路径和累加的结果。即dp的第一行和第一列值如下：

[1491891422]\begin{bmatrix} 1 & 4 & 9 & 1 & 8\\ 9 \\ 14 \\ 22 \end{bmatrix} ​⎣​⎢​⎢​⎡​​​1​9​14​22​​​4​​​9​​​1​​​8​​​⎦​⎥​⎥​⎤​​

除第一行和第一列的其他位置(i,j)外，都有左边位置(i-1,j)和上边位置(i,j-1)。从(0,0)到(i,j)的路径必然经过位置(i-1,j)或位置(i,j-1)，所以，dp[i][j] = min{dp[i-1][j],dp[i][j-1]}+m[i][j]。代码如下：

public int minPathSum(int[][] m) {
    if (m == null || m.length == 0) {
        return 0;
    }
    int[][] dp = new int[m.length][m[0].length];
    return process(m, m.length - 1, m[0].length - 1, dp);
  }

public int process(int[][] m, int row, int col, int[][] dp) {
    dp[0][0] = m[0][0];
    for (int i = 1; i < m.length; i++) {
        dp[i][0] = dp[i - 1][0] + m[i][0];
    }
    for (int j = 1; j < m[0].length; j++) {
        dp[0][j] = dp[0][j - 1] + m[0][j];
    }
    for (int i = 1; i < m.length; i++) {
        for (int j = 1; j < m[0].length; j++) {
            dp[i][j] = Math.min(dp[i][j - 1], dp[i - 1][j]) + m[i][j];
        }
    }
    return dp[dp.length - 1][dp[0].length - 1];
}

TIP

矩阵中一共有MxN的位置，每个位置都计算一次，从(0,0)位置到达自己的最小路径和，计算的时候只是比较上边位置的最小路径和与左边位置的最小路径和哪个更小，所以时间复杂度为 O(M×N)\ O(M\times N)O(M×N)，dp矩阵的大小为M×N\ M\times NM×N，所以额外空间复杂度为O(M×N)\ O(M\times N)O(M×N)。

# 动态规划方法压缩空间法

  1. 在动态规划方法解答中，对于m的第一行所有位置来说，即m[0，0..N-1]，从m[0，0]走到m[0,j]位置的路径和就是m[0][0..j]的所有位置的最小路径和累加的结果。 即dp的初始值如下：

[14918]\begin{bmatrix} 1 & 4 & 9 & 1 & 8 \end{bmatrix} [​1​​​4​​​9​​​1​​​8​​]

  1. 步骤1中dp[j]表示从[0,0]到[0,j]位置的最小路径和，在这一步中想把dp[j]更新成从[0,0]到[1,j]位置的最小路径和，首先将dp[0]更新为[0,0]到[1,0]位置的最小路径和,即 dp[0] = dp[0]+m[1][0]，然后让dp[1]代表从[0,0]位置到[1][1]位置的最小路径和，即dp[1] = min { dp[0] , dp[1] }+m[1][1]，同理，dp[2] = min{ dp[1] , dp[2] } + m[1][2]。
  2. 重复步骤2的更新过程，一直到m矩阵的最后一行。
    代码如下：
public int minPathSum(int[][] m) {
    if (m == null || m.length == 0) {
        return 0;
    }
    return process(m, m.length - 1, m[0].length - 1);
}

public int process(int[][] m, int row, int col) {
    int[] dp = new int[m[0].length];
    dp[0] = m[0][0];
    for (int j = 1; j < m[0].length; j++) {
        dp[j] = dp[j - 1] + m[0][j];
    }
    for (int i = 1; i < m.length; i++) {
        dp[0] =  dp[0]+ m[i][0];
        for (int j = 1; j < m[0].length; j++) {
            dp[j] = Math.min(dp[j - 1], dp[j]) + m[i][j];
        }
    }
    return dp[dp.length - 1];
}

TIP

动态规划经过空间压缩后的方法，虽然时间复杂度依然是O(M×N)\ O(M\times N)O(M×N)，但是额外空间复杂度可以从O(M×N)\ O(M\times N)O(M×N)减小至O(N)\ O(N)O(N)，也就是不使用大小为O(M×N)\ O(M\times N)O(M×N)的空间，而是仅仅使用大小为O(N)\ O(N)O(N)的空间。
