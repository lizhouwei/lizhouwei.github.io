# 题目

有n个台阶的楼梯，每次只能上 1个 或者 2个台阶，那么到达顶端共有多少种不同的方法？

# 解析

  • 最终问题f(i)：
    假设从0到达第i个台阶的方法共有f(i)个。
  • 最优子结构f(i-1)，f(i-2)：
    到达i个台阶，有两种可能，第一种可能是从第 i-1 个台阶上1个台阶到达终点，第二种可能是从第 i-2 个台阶上2个台阶到达终点。
  • 最优子结构与最终问题之间的关系：
    按照上述表达，那么可以归纳出f(i) = f(i-1) + f(i-2) (i>=3)
    并且如果台阶只有1个，则方法数只有1，即f(1)=1 ; 如果台阶只有2个，则方法数只有2个，即f(2)=2;所以结束条件为f(1) = 1,f(2) = 2,完整公式如下：

# 暴力搜索方法解答

public int sum(int n){
  if(n <= 0){
    return 0;
  }
  if( n == 1 || n == 2 ){
    return n;
  }
  return sum(n-1) + sum(n-2);
}

TIP

递归的时间复杂度是由阶梯数和最优子结构的个数决定的。这里的阶梯数是 N ，最优子结构个数是2 。如果想象成一个二叉树，那么就可以认为是一个高度为N-1，节点个数接近 2 的 N-1 次方的树，因此此方法的时间复杂度可以近似的看作是 O(2n)\ O(2n)O(2n)。

# 记忆搜索方法解答

public int sum(int n){
 
  int[] map = new int[n];
  return process(  n,  map) ;
}
public int process(int n,int[] map){
  if(n <= 0){
    return 0;
  }
  if( n == 1 || n == 2 ){
    return n;
  }
  int mapValue = map[n];
  if(mapValue !=0){
    return mapValue == -1 ? 0 : mapValue;
  }
  int res = process(n-1,map) + process(n-2,map);
  map[n] = res == 0 ? -1 : res;
  return res;
}

TIP

这里哈希表里存了 n+1 个结果，时间复杂度和空间复杂度都是O(n)\ O(n)O(n)。程序性能得到了明显优化

# 动态规划方法解答

public int sum(int n){
 if(n <= 0){
    return 0;
  }
  if( n == 1 || n == 2 ){
    return n;
  }
  return process( n ) ;
}
public int process(int n){
  int[] dp = new int[n+1];//dp表示当有n个台阶时的方法数
  dp[1] = 1;
  dp[2] = 2;
  for(int j = 3 ; j<= n ;j++){
    dp[j] =dp[j-1]+dp[j-2];
  }
  return dp[n];
}
